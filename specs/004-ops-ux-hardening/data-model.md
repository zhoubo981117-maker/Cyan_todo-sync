# Phase 1 数据模型：运维与体验加固

本特性**不引入任何新数据表，也不修改既有表结构**（FR-010/011）。下列为涉及的概念实体，仅用于说明取值来源与优先级，非数据库 schema。

## 实体：版本标识（Version Identifier）

线上当前运行的构建版本，用于页面展示与"是否有更新"的判断。

| 属性 | 含义 | 取值来源（按优先级） |
|------|------|----------------------|
| `commit` | 完整提交标识 | `TODO_APP_VERSION` → `VERCEL_GIT_COMMIT_SHA` → 本地 `git rev-parse HEAD` → `unknown` |
| `short` | 短标识（前 7 位），用于展示、`localStorage` 比较与 SW 缓存名 | 同上，取前 7 位 |
| `source` | 来源标注（可观测性） | `env` / `vercel` / `git` / `unknown` |

- 消费方：`/api/version`（应用内提示）、`/sw.js` 占位符替换（缓存刷新）。
- 状态转移：每次部署 `commit` 变化 → `short` 变化 → 触发应用内提示 + SW 缓存换名清旧。
- 校验规则：任何来源缺失时静默回退，不得抛错（边界场景）。

## 实体：签名密钥（Signing Secret）

用于校验登录凭证（HMAC 签名）的密钥。

| 属性 | 含义 | 取值来源（按优先级） |
|------|------|----------------------|
| 值 | 32 字节密钥 | `TODO_SIGNING_SECRET`（env，最高）→ Postgres 模式 `app_kv` 表持久化 → 本地文件 `secret.key` |

- 既有存储：`app_kv` 表（已存在，本特性不改其结构）。
- 优先级语义：env 存在时**始终**优先，使密钥独立于数据库内容；清库/换库后只要 env 稳定即不掉登录。
- 不变量：未配置 env 时维持现有"自动生成并持久化"行为，无回归（FR-007）。
- 本特性不新增、不迁移、不删除任何密钥数据。
